## Database Structure

### device_tokens Table
```
id               | UUID (auto)
user_id          | TEXT (John's user ID)
family_group_id  | TEXT (Smith Family)
fcm_token        | TEXT (long token string)
platform         | TEXT ("android" or "ios")
created_at       | TIMESTAMP
updated_at       | TIMESTAMP
```

### urgent_items Table
```
id               | TEXT (UUID)
name             | TEXT ("Milk")
family_group_id  | TEXT (Smith Family)
created_by       | TEXT (Sarah's user ID)
created_by_name  | TEXT ("Sarah")
created_at       | BIGINT (timestamp)
resolved_by      | TEXT (John's user ID or NULL)
resolved_by_name | TEXT ("John" or NULL)
resolved_at      | BIGINT (timestamp or NULL)
price            | NUMERIC (2.50 or NULL)
status           | TEXT ("active" or "resolved")
sync_status      | TEXT ("synced" or "pending")
updated_at       | TIMESTAMP
```

## Key Points

1. **Offline-First**: Item saved locally first, syncs when online
2. **True Push**: FCM works even when app is killed
3. **No Notification Loop**: Only creator's action triggers notification
4. **Budget Integration**: Resolved items with prices auto-added to budget
5. **Real-time Updates**: All devices stay in sync via Supabase
6. **Free**: 100% free using Supabase Edge Functions + FCM free tier

## Notification Behavior

|-------------------|----------------------------------------------------------|
